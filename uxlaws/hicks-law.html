Hick’s Law

Hick’s Law states that the time it takes for a user to make a decision increases with the number and complexity of choices.

This means that more options lead to harder decisions and slower response times. As designers, we should aim to simplify interfaces by reducing choices or grouping them logically to improve user decision speed and satisfaction.

Case Study

For example, Amazon’s filtering system allows users to narrow down thousands of products step-by-step, which helps avoid overwhelming the user with too many options at once.

Application

In my project, I implemented Hick’s Law by using progressive disclosure and categorized filters so users could find products more efficiently without feeling overwhelmed.
